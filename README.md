# E2E Test Automation - 두부핑퐁

## 개요
두부핑퐁 관리 대시보드(`ppistaging.dubuhealth.in`)의 **End-to-End 테스트 자동화** 프로젝트

- **프레임워크**: Playwright + TypeScript
- **레포**: https://github.com/dubu-harry/e2e-automation

## 실행 방법

```bash
npm install
npx playwright install chromium
cp .env.example .env  # 계정 정보 입력

npx playwright test --project=e2e                       # E2E 테스트
npx playwright test --project='배포전TC-collaboration'   # 배포전TC
npx playwright show-report                              # 리포트
```

## E2E Test Cases

> 모든 테스트는 로그인 완료 후 (인증된 세션 상태) 실행됩니다.
> 멀티탭 테스트는 `browser.newContext()`로 호스트/게스트 세션을 완전 분리합니다.

| EDGE-04-02 | 이름만 입력 | 이름 "테스트" 입력, 전화번호/인덱스 비어있음 | "입장하기" 버튼 상태 확인 (비활성화 또는 서버 검증) | MID |
| EDGE-04-03 | 존재하지 않는 인덱스 | 이름 "테스트", 전화번호 "0000", 인덱스 "99999" 입력 → 입장 시도 | 입장 실패 또는 에러 처리 (세션 화면 미진입) | HIGH |
| EDGE-04-04 | 전화번호에 문자 입력 | 전화번호 필드에 "abcd" 입력 | 숫자만 허용하거나, 입력되더라도 크래시 없음 | MID |
| EDGE-04-05 | 음수 인덱스 | 이름 "테스트", 전화번호 "1234", 인덱스 "-1" 입력 | 에러 처리 또는 비활성화 (크래시 없음) | LOW |
| EDGE-04-06 | 소수점 인덱스 | 인덱스에 "1.5" 입력 | 에러 처리 또는 정수 변환 (크래시 없음) | LOW |

### EDGE-05. 세션 Edge Cases (멀티탭)

| ID | 테스트 케이스 | 수행 테스트 | 기대 결과 | 우선순위 |
|---|---|---|---|---|
| EDGE-05-01 | 호스트 새로고침 | 세션 진행 중 호스트 페이지 `F5` 새로고침 | 세션 유지 또는 재연결 시도 (페이지 크래시 없음) | HIGH |
| EDGE-05-02 | 모달 오버레이 처리 | 세션 중 모달이 표시된 상태에서 ESC/확인 | 모달 정상 닫힘, 세션 계속 진행 | MID |
| EDGE-05-03 | 수업 종료 다이얼로그 | "수업 종료" 클릭 → "회기 종료" 모달 확인 → 취소 클릭 | 완료/중단/취소 3개 옵션 표시, 취소 시 다이얼로그 닫히고 세션 유지 | HIGH |
| EDGE-05-04 | 게스트 새로고침 | 세션 진행 중 게스트 페이지 `F5` 새로고침 | 비디오 재표시 또는 재진입 화면 (크래시 없음) | MID |

### EDGE-06. 페이지 안정성 Edge Cases

## 프로젝트 구조

```
e2e-automation/
├── src/
│   ├── config/          # 환경설정, 라우트, 셀렉터
│   ├── fixtures/        # Playwright fixtures (auth, collaboration)
│   ├── pages/           # Page Object Model
│   └── types/           # TypeScript 타입
├── tests/
│   ├── auth/            # 로그인 setup (storageState)
│   ├── e2e/             # E2E 테스트 (20개 파일, 115 TC)
│   ├── edge-cases/      # Edge Case 테스트 (7개 파일, 35 TC)
│   └── 배포전TC/        # 기존 스모크 테스트 보관 (56 TC)
├── playwright.config.ts
└── .env.example
```

```bash
# 전체 실행

# 배포전TC만
npx playwright test --project='배포전TC-navigation' --project='배포전TC-pages' --project='배포전TC-session' --project='배포전TC-collaboration'
```
